# إصلاح مشكلة إدارة الموظفين - دليل الاستخدام

## المشكلة
- الـ dropdown في صفحة StaffManagement يظهر بخلفية سوداء
- لا يمكن إضافة موظفين جدد
- الدوال تجلب البيانات من جدول `profiles` بدلاً من `users`

## الحل

### الخطوات:

#### 1. تطبيق إصلاح قاعدة البيانات
اذهب إلى Supabase SQL Editor وقم بتطبيق الملف:
```
database/complete_staff_fix_updated.sql
```

#### 2. التحقق من تطبيق الإصلاحات
- ستحصل على رسالة تأكيد تحتوي على أسماء المستخدمين التجريبيين
- تأكد من وجود المستخدمين في جدول `users`

#### 3. تشغيل المشروع
```bash
npm run dev
```

#### 4. اختبار النظام
1. اذهب إلى صفحة تسجيل الدخول
2. سجّل دخول كمالك مطعم
3. اذهب إلى قسم "إدارة الموظفين"
4. اضغط "إضافة موظف"
5. اختر أحد المستخدمين من القائمة (مثل "أحمد محمد")
6. حدد الصلاحيات المطلوبة
7. اضغط "إضافة"

## ما تم إصلاحه

### في قاعدة البيانات:
- ✅ إضافة عمود `full_name` إلى جدول `users`
- ✅ إنشاء جدول `staff_permissions`
- ✅ إضافة بيانات تجريبية للمستخدمين
- ✅ إعداد RLS policies بسيطة

### في الكود:
- ✅ تعديل `getAvailableUsersForStaff` لتجلب من `users` بدلاً من `profiles`
- ✅ تعديل `getAllStaffPermissions` لتجلب من `users`
- ✅ تعديل `updateStaffPermissions` لتجلب البريد من `users`
- ✅ إضافة console.log للـ debugging

## استكشاف الأخطاء

### إذا لم يظهر المستخدمون في الـ dropdown:
1. افتح Developer Tools (F12)
2. اذهب إلى Console
3. ابحث عن الرسائل التي تحتوي على "Loading available users" و "Available users loaded"
4. تأكد من أن البيانات تُجلب بشكل صحيح

### إذا ظهرت أخطاء:
1. تأكد من تطبيق ملف SQL
2. تأكد من وجود بيانات في جدول `users`
3. تحقق من console للأخطاء

## النتيجة المتوقعة

- تعديل صلاحيات الموظفين الموجودين
- حذف الموظفين عند الحاجة
