## الخطوات للتطبيق

### 1. تطبيق الإصلاحات في قاعدة البيانات

اذهب إلى **Supabase SQL Editor** وقم بتطبيق الملف:
```
database/final_staff_fix_complete.sql
```

### 2. إضافة مستخدم حقيقي

#### الطريقة الأولى: من قاعدة البيانات
```sql
-- استبدل البيانات ببياناتك الحقيقية
INSERT INTO users (
  id,
  email,
  password_hash,
  full_name,
  role,
  is_active,
  temp_password
) VALUES (
  gen_random_uuid(),
  'your-email@example.com',        -- بريدك الإلكتروني الحقيقي
  crypt('your-password', gen_salt('bf', 8)),  -- كلمة مرورك الحقيقية
  'اسمك الحقيقي',                  -- اسمك الحقيقي
  'staff',                        -- الدور (staff/owner/manager)
  true,                           -- تفعيل الحساب
  false                           -- كلمة مرور دائمة
);
```

#### الطريقة الثانية: من خلال النظام
1. اذهب إلى صفحة التسجيل: `/register`
2. أدخل بيانات المستخدم الجديد
3. سجّل دخولك كمالك مطعم
4. اذهب إلى إدارة الموظفين
5. أضف المستخدم كموظف

### 3. اختبار النظام

#### اختبار إضافة موظف:
1. اذهب إلى `/restaurant/staff`
2. اضغط "إضافة موظف"
3. اختر مستخدم من القائمة
4. حدد الصلاحيات
5. اضغط "إضافة"

#### اختبار حذف موظف:
1. اضغط على زر "حذف" بجانب الموظف
2. أكد الحذف

#### اختبار تسجيل الدخول:
1. اذهب إلى `/login`
2. أدخل بريد الموظف وكلمة المرور
3. تأكد من تسجيل الدخول بنجاح

## استكشاف الأخطاء

### إذا ظهرت أخطاء في Console:

#### خطأ في إضافة الموظف:
```
Error updating staff permissions: {...}
```
**الحل:** تأكد من تطبيق ملف SQL وتعطيل RLS

#### خطأ في حذف الموظف:
```
Error removing staff permissions: {...}
```
**الحل:** نفس الحل أعلاه

#### خطأ في تسجيل الدخول:
```
فشل تسجيل الدخول
```
**الحل:** تأكد من صحة البريد وكلمة المرور في جدول users

### فحص البيانات:

#### لفحص المستخدمين:
```sql
SELECT id, email, full_name, role FROM users WHERE is_active = true;
```

#### لفحص الموظفين:
```sql
SELECT * FROM staff_permissions;
```
